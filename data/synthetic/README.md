# Synthetic Schema Sample (NOT operational data)

`traffic_has_linked_threat_synthetic.csv` is a **100-row, fully synthetic**
illustration of the input structure expected by this pipeline. It exists so
that readers and reviewers can inspect the exact column layout and value style
of the processed dataset, and smoke-test the code paths, without any
operational telemetry being released.

## Why synthetic instead of a real anonymised sample

The source telemetry belongs to an institutional firewall environment. The
manuscript's data-governance statement requires explicit institutional
authorization and a privacy/security review before **any** row-level release,
including small anonymised excerpts, because even processed rows expose
operational context (zone, interface, application, and country distributions)
and can carry residual re-identification risk when linked with external
information. A synthetic sample provides the practical benefits of an excerpt
(visible schema, runnable scripts) with zero disclosure risk, which is why it
was chosen as the release mechanism.

## Provenance guarantees

- Generated by `code/make_synthetic_sample.py` with a fixed seed (42); rerun it
  to reproduce this exact file.
- No value is drawn from the operational corpus. Value domains are plausible
  vendor-style values consistent with the aggregate profiles already published
  in `results/` (for example, the application and session-end-reason families).
- Timestamps use a deliberately fake base date (2000-01-01), session
  identifiers are sequential `synthetic-*` keys, and rule names are
  `synthetic-rule-*`.
- The positive share is 5/100, intentionally higher than the real 1.84%
  natural prevalence, so that stratified train/test splits remain runnable on
  a 100-row file. **Nothing computed from this file reproduces any number
  reported in the manuscript.**

## Smoke test

```bash
python code/make_synthetic_sample.py                # regenerate
# in a scratch copy of the repo:
cp data/synthetic/traffic_has_linked_threat_synthetic.csv \
   data/processed/traffic_has_linked_threat.csv
python code/03_benchmark_baseline.py --dataset linked_full --feature-set core
```

The benchmark loader will read the file, exclude the identifier/time/leakage
columns (`Session ID`, `session_key`, `Receive Time`, offsets, `Rule`,
`Action Source`, `Type`, `Threat/Content Type` raw fields per the script's
exclusion lists), and train on the 27 core features. Results on 100 synthetic
rows are meaningless by design; the run only demonstrates that the pipeline
executes end to end on the documented schema.

Real row-level processed artifacts remain available only under the conditions
described in `data/processed/README.md`.
